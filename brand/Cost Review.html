// 04 — Notes from the FinOps lead

findings → priority actions for next month
// finding 01
Coverage is the
biggest lever.
You're 62% covered against a target of ≥80%. Buying the AWS Compute SP and Azure SQL RI alone moves you to 78% and unlocks £339k/yr — before a single workload is changed.
// finding 02
SQL is over-
provisioned.
Aurora and Azure SQL are sized for peak that lasts ~4 hours/day. Reserved capacity + Hybrid Benefit pulls steady-state cost down by ~44% with no perf impact on the workload profile we modelled.
// finding 03
Don't
over-commit.
We've sized commitments to ~85% of the 90-day rolling baseline. The remaining 15% stays on-demand to absorb the seasonality you flagged — Black Friday + year-end batch.
