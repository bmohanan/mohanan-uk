// 01 — executive summary

The headline:
between 22% and 30% of annual
cloud spend is recoverable.

Northwind currently spends approximately £285k/month across AWS and Azure. From the data-room sample, three things are clear: commitment coverage is low, SQL is over-provisioned, and the platform's growth has out-paced the FinOps practice that should be matching it.

BMC has run this engagement fourteen times in the last two years. The median outcome is a 28% reduction in steady-state cloud cost, landed within six weeks of kickoff, with a commercial payback of under five months on the advisory fee.

// today

£285k / mo
Combined AWS + Azure run-rate. 62% commitment coverage against an ≥80% target. Three workstreams flagged as over-provisioned by Cost Explorer / Azure Advisor.

// after BMC

£213k / mo
Target steady-state after the six-week engagement. £71k/month of identified savings landed; 78% coverage; runbook handed to platform team.

This proposal sets out the approach, scope, commercial terms and timeline. The accompanying cost-review worksheet (BMC-FN-2026-05-001) shows the line-item evidence behind the numbers above.

// 02 — approach

Six weeks. Three phases. No parallel governance.

// PHASE 01 · WEEKS 1–2

Read the bill.

Baseline the trailing 90 days. Map workloads to services. Surface the top-20 lines.

  • CUR + Azure exports ingested
  • Tag-coverage audit
  • Unit-economics baseline
// PHASE 02 · WEEKS 3–4

Land the easy wins.

Right-size, lifecycle, retire. Anything that doesn't need a committed-use plan, ship now.

  • VM & EC2 right-size
  • S3 / Blob lifecycle
  • Orphan-resource cleanup
// PHASE 03 · WEEKS 5–6

Lock the savings.

Buy the Savings Plans / RIs / Hybrid Benefit sized to ≤85% of the new baseline. Hand over.

  • 3-yr SP & RI purchase
  • Azure Hybrid Benefit
  • Runbook + monthly review
